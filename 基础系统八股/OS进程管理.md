#  **操作系统核心面试知识点总结：进程、线程与调度**

## 一、进程（Process）

### 1. 概念与特点

- **定义**：一个程序在某个数据集上的一次执行，是系统进行资源分配和调度的基本单位。
- **特点**：
  1. **动态性**：进程是程序的执行过程，有创建、运行、暂停、结束等生命周期。
  2. **独立性**：每个进程有自己独立的地址空间。
  3. **并发性**：多个进程可以“同时”存在（宏观并行，微观交替执行）。

> 💡 程序是静态的代码文件；进程是运行中的程序。

------

### 2. 进程状态与转换

常见五种状态：

| 状态                | 说明                                           |
| ------------------- | ---------------------------------------------- |
| **新建（New）**     | 进程刚被创建，正在初始化。                     |
| **就绪（Ready）**   | 所需资源齐全，等待CPU调度执行。                |
| **运行（Running）** | 占用CPU正在执行。                              |
| **阻塞（Blocked）** | 等待某事件（如I/O完成），即使给CPU也无法执行。 |
| **结束（Exit）**    | 执行完毕或异常退出。                           |

扩展状态（虚拟内存系统中）：

- **就绪挂起**：进程在磁盘上等待执行。
- **阻塞挂起**：进程在磁盘上等待事件完成。

#### 状态转换示例：

- New → Ready：初始化完成。
- Ready → Running：调度器分配CPU。
- Running → Blocked：I/O请求。
- Blocked → Ready：I/O完成。
- Running → Ready：时间片耗尽。
- Running → Exit：执行完毕或异常终止。

------

### 3. 进程控制块（PCB）

- 操作系统用来描述和控制进程运行的核心数据结构。
- **内容包括**：
  - 进程标识信息：PID、父进程号。
  - 进程状态：就绪、阻塞等。
  - CPU寄存器内容（程序计数器、堆栈指针等）。
  - 资源清单：打开的文件、I/O设备、内存映射。
  - 调度信息：优先级、时间片、CPU使用时间等。
- **作用**：
  - 系统根据PCB管理所有进程；
  - 各状态进程通过PCB链表组成队列（就绪队列、阻塞队列）。

------

### 4. 进程控制操作

| 操作     | 说明                                          |
| -------- | --------------------------------------------- |
| **创建** | 操作系统为新进程分配PCB、内存空间和必要资源。 |
| **终止** | 释放进程资源，更新状态表。                    |
| **阻塞** | 进程等待事件，切换状态并加入阻塞队列。        |
| **唤醒** | 事件完成，从阻塞队列移至就绪队列。            |

> 子进程终止 → 资源交还父进程；父进程终止 → 子进程被1号进程（init）接管。

------

### 5. 进程上下文切换

- **上下文**：CPU执行任务所需的寄存器状态、程序计数器、内核栈等信息。
- **切换过程**：
  1. 保存当前进程的上下文到其PCB；
  2. 加载下一个进程的上下文到CPU；
  3. 跳转执行新进程。
- **切换开销**：
  - 涉及内核态操作；
  - 切换虚拟地址空间；
  - CPU cache失效。

**触发场景**：

- 时间片用完；
- I/O阻塞；
- 调用sleep；
- 有更高优先级的任务；
- 硬件中断发生。

------

## 二、线程（Thread）

### 1. 定义与作用

- **线程**是CPU调度的最小单位，是进程中独立的执行流。
- 同一进程内的多个线程共享代码段、堆、全局变量等资源，但各自拥有独立的**寄存器和栈**。

### 2. 优缺点

| 优点                                                         | 缺点                                                         |
| ------------------------------------------------------------ | ------------------------------------------------------------ |
| - 创建、销毁速度快； - 切换开销小； - 线程间通信无需内核，效率高。 | - 共享内存易产生竞态条件； - C/C++中线程崩溃可能导致整个进程崩溃。 |

------

### 3. 进程 vs 线程

| 对比项     | 进程                            | 线程                   |
| ---------- | ------------------------------- | ---------------------- |
| 资源分配   | 独立的地址空间                  | 共享进程资源           |
| 调度单位   | 是资源分配的基本单位            | 是CPU调度的基本单位    |
| 创建与销毁 | 慢，需要分配资源                | 快，仅需创建栈与寄存器 |
| 切换开销   | 高，需要切换页表、虚拟内存      | 低，不需切换页表       |
| 通信方式   | IPC（管道、共享内存、消息队列） | 直接读写共享内存       |

------

### 4. 线程上下文切换

- 同进程内线程切换：只需切换寄存器、程序计数器和栈。
- 不同进程的线程切换：需额外切页表、虚拟内存、内核栈。

> ✅ 因此线程上下文切换比进程轻量。

------

## 三、线程实现模型

| 模型                         | 描述                                     | 优点                           | 缺点                                     |
| ---------------------------- | ---------------------------------------- | ------------------------------ | ---------------------------------------- |
| **用户线程 (User Thread)**   | 在用户空间实现，由线程库管理。OS不可见。 | 创建和切换快；不需内核态切换。 | 一个线程阻塞，全进程阻塞；不能利用多核。 |
| **内核线程 (Kernel Thread)** | 由内核管理和调度。                       | 阻塞互不影响；支持多核并行。   | 系统调用频繁，开销大。                   |
| **轻量级进程 (LWP)**         | 用户线程与内核线程的中间层。             | 综合两者优点，可灵活映射。     | 实现复杂，需额外维护。                   |

三种映射模型：

- **1:1**：每个用户线程对应一个内核线程。
- **N:1**：多个用户线程映射到一个内核线程。
- **M:N**：多个用户线程映射到多个内核线程（折中方案）。

## 四、调度（Scheduling）

### 1. 调度时机

- 进程从 Ready → Running；
- Running → Blocked；
- Running → Exit；
- 时间片到期；
- 中断发生。

### 2. 调度目标（五大指标）

| 指标           | 说明                                 |
| -------------- | ------------------------------------ |
| **CPU利用率**  | CPU尽可能忙碌，提高效率。            |
| **系统吞吐量** | 单位时间内完成的进程数。             |
| **周转时间**   | 等待+执行+I/O，总时间越短越好。      |
| **等待时间**   | 进程在就绪队列中等待的时间越短越好。 |
| **响应时间**   | 用户请求到系统第一次响应的时间。     |

## 五、调度算法详解

| 算法                     | 类型        | 思想                         | 优点                       | 缺点                   | 适用场景   |
| ------------------------ | ----------- | ---------------------------- | -------------------------- | ---------------------- | ---------- |
| **FCFS**（先来先服务）   | 非抢占      | 按到达顺序执行               | 公平、简单                 | 长作业影响短作业       | CPU密集型  |
| **SJF**（最短作业优先）  | 非抢占      | 优先运行执行时间最短的任务   | 平均等待时间短             | 长作业饥饿；难预估时间 | 批处理     |
| **HRRN**（高响应比优先） | 非抢占      | 响应比 = (等待+服务)/服务    | 兼顾长短作业               | 难实际实现             | 理想算法   |
| **RR**（时间片轮转）     | 抢占        | 每个进程分配固定时间片       | 公平，适合交互系统         | 时间片设太短效率低     | 多任务系统 |
| **HPF**（最高优先级）    | 抢占/非抢占 | 优先级高者先执行             | 适用于实时任务             | 低优先级任务可能饥饿   | 实时系统   |
| **MLFQ**（多级反馈队列） | 抢占        | 多队列+时间片+动态优先级调整 | 兼顾短作业响应与长作业公平 | 实现复杂               | 通用系统   |

### 多级反馈队列（MLFQ）机制：

1. 多个队列优先级从高到低；
2. 新进程进入最高优先级队列；
3. 若时间片用完未完成 → 下降到下一级；
4. 低级队列仅在高级为空时调度；
5. 高级进程可随时抢占。

> ✅ 兼顾交互性（短任务）与公平性（长任务）。

## 六、面试核心问答模板

| 面试问题                       | 推荐回答                                                     |
| ------------------------------ | ------------------------------------------------------------ |
| **进程与线程的区别？**         | 进程是资源分配单位，线程是调度单位。进程有独立地址空间，线程共享资源但有独立栈和寄存器。线程创建和切换比进程快。 |
| **为什么要有线程？**           | 提高并发和CPU利用率。进程间通信慢，线程共享内存可快速通信。  |
| **上下文切换是什么？**         | 保存当前任务CPU状态并加载下一个任务状态，让不同任务共享CPU。 |
| **线程切换与进程切换区别？**   | 线程切换不涉及页表切换；进程切换需要保存/恢复虚拟内存和内核栈。 |
| **时间片机制的意义？**         | 防止某一进程长期占用CPU，保证公平性和系统响应。              |
| **时间片大小怎么选？**         | 通常20–50ms，太短→切换频繁；太长→响应差。                    |
| **多级反馈队列的优点？**       | 同时兼顾响应时间与吞吐率。短作业快执行，长作业逐级下降避免饥饿。 |
| **内核线程与用户线程的区别？** | 用户线程由线程库管理，不可被内核调度；内核线程由内核直接调度，支持多核并发。 |

# 🧩 操作系统面试重点（二）：进程间通信（IPC）

## 一、为什么需要进程间通信？

> 每个进程都有**独立的用户地址空间**，无法直接访问彼此的内存；
>  但它们**共享同一个内核空间**，所以通信必须经过内核来“中转”数据。

通俗来说：

> “用户空间之间不通气，只有通过内核这条‘地下管道’才能传话。”

Linux 为此提供了多种 IPC 机制，按复杂度和应用场景大致分为：

| 分类           | 名称              | 关键特性                           |
| -------------- | ----------------- | ---------------------------------- |
| **基础型**     | 管道（匿名/命名） | 最早期的 IPC，单向流式通信         |
| **消息型**     | 消息队列          | 结构化消息，异步通信               |
| **共享内存型** | 共享内存 + 信号量 | 高速通信、需同步机制保护           |
| **异步信号型** | 信号              | 异步事件通知机制                   |
| **网络型**     | Socket            | 可跨主机通信，支持 TCP/UDP/UNIX 域 |

------

## 二、管道（Pipe）

### 1️⃣ 概念

管道是一种半双工通信机制：

> 数据**单向流动**，遵循先进先出（FIFO）原则。

- 数据写入管道缓冲区（内核空间）
- 另一端从管道中读取
- 通信完成后自动销毁

### 2️⃣ 区分类型

| 类型                 | 特点                                     | 场景                           |
| -------------------- | ---------------------------------------- | ------------------------------ |
| **匿名管道（pipe）** | 无文件名，只能在**父子进程**之间通信     | Shell 命令 `ps aux             |
| **命名管道（FIFO）** | 有路径名，可在**无亲缘关系的进程**间通信 | `mkfifo myPipe` 创建的设备文件 |

匿名管道存在于内存中；命名管道存在于文件系统中（类型 `p`）。

### 3️⃣ 创建与原理

```
int pipe(int fd[2]); // fd[0] 读端，fd[1] 写端
```

- `fork()` 后子进程继承父进程文件描述符 → 达到 IPC 效果。
- 若需双向通信：需建立两条管道（各一方向）。

### 4️⃣ 优缺点

✅ 优点：简单易用、同步机制天然（读写阻塞）。
 ❌ 缺点：

- 只能单向；
- 仅限父子进程；
- 传输效率低（内核缓冲受限）；
- 无格式字节流。

------

## 三、消息队列（Message Queue）

### 1️⃣ 原理

- 消息队列是**内核中维护的消息链表**。
- 每条消息称为一个**消息体（message）**，用户可自定义结构。
- 消息体存储于内核，接收方按类型取出，读取后删除。

### 2️⃣ 特点

- **异步通信**：发送方放入队列即可返回；接收方稍后取。
- 数据有结构（不像管道字节流）。
- 生命周期随内核，不随进程结束而消失。
- 可用于**多对多通信**（多个进程读写同一队列）。

### 3️⃣ 限制

- 有消息体大小上限（`MSGMAX`），队列总容量上限（`MSGMNB`）。
- 每次收发都涉及**用户态 ↔ 内核态**拷贝两次，性能不如共享内存。

### 4️⃣ 优缺点

✅ 优点：

- 结构化通信，灵活。
- 异步通信，无需阻塞等待。

❌ 缺点：

- 内核拷贝多次，效率低于共享内存。
- 不适合大数据传输。
